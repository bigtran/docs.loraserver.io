LoRa Server, 开源的 LoRaWAN 网络服务器（network-server）

CableLabs ® 赞助

LoRa Server 项目提供构建LoRaWAN网络的开源组件。提供了一个可直接部署使用的解决方案，包括用户友好的web界面，gRPC 和 REST 接口。这些组件可以定制化，或者集成到现有的硬件里。所有组件是MIT授权，可以商用。

功能

支持Class-A

Class A 节点，允许双向通信，每一个上行通信跟着两个短的下行窗口。终端节点的传输时隙（transmission slot）基于其自身通信需要有一个随机时间变化（ALOHA类型的协议）。

支持Class-B

Class B 节点，允许更多的接受时隙（recieve slot），除了Class A节点的随机接受窗口，Class B节点在特定时间还会打开接受串口。为了让节点在预定的时间打开接受窗口，需要节点和网关（基站）之间做时间信标（time synchronized Beacon）同步。

支持Class-C

Class C 节点，基本上是连续的打开接收窗口，仅在发送时关闭，所以比Class A和Class B类节点耗电，适合常供电类的场景。但他们为服务器和终端节点提供最低延迟的通信。

自适应的data-rate

当终端设备启用了ADR（adaptive data-rate）时，LoRa Server将确保设备使用最理想的 数据速率（data-rate）和发射功率（tx-power）。 不仅可以节省节点的功耗，还能优化无线电频谱的使用，降低碰撞风险。

实时帧记录（frame-logging）

LoRa (App) Server 应用服务器为每个网关和节点提供实时帧记录，显示所有的 发送和接收 元数据、可读的 LoRaWAN PHYPayload原始数据，类似LoRaWAN的抓包工具。

频道配置、充配置

不管是否要使用LoRaWAN定义的通道的子集（比如美国频段），还是想要配置其他通道（如欧盟频段），LoRa Server 会确保终端节点和网络保持通道的同步 (使用 CFList 字段或mac命令)。

多租户 Multi-tenant

LoRa App Server 应用服务器支持托管可以分配用户（管理员）的多个不同组组织。使用 PBKDF2密码，LoRa 应用服务器可以和MQTT代理集成 身份验证和授权，这样不同组织就可以看到他们自己的节点数据。

接口与集成

所有组件提供 gRPC 或 REST接口，与外部服务集成。默认情况下，所有的应用数据通过MQTT通道发布，每个应用的数据、事件都可以设定HTTP端点（endpoint）。

LoRaWAN 1.0/1.1适配

LoRa Server 支持 LoRaWAN 1.0 和 LoRaWAN 1.1 设备，包括所有区域（国家）参数规范和带宽。
